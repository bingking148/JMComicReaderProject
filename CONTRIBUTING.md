# 贡献指南

感谢你对 JM漫画阅读器 项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告 Bug

如果你发现了 Bug，请：

1. 在 [Issues](https://github.com/yourusername/JMComicReaderProject/issues) 页面搜索，确认问题未被报告
2. 如果是新问题，点击 "New Issue"
3. 选择合适的模板（Bug Report）
4. 填写详细信息：
   - Bug 的标题和描述
   - 复现步骤
   - 期望行为
   - 实际行为
   - 截图（如果适用）
   - 环境信息（操作系统、Python版本等）

### 提出新功能

如果你有好的想法：

1. 在 [Issues](https://github.com/yourusername/JMComicReaderProject/issues) 页面搜索，确认功能未被建议
2. 如果是新建议，点击 "New Issue"
3. 选择 "Feature Request" 模板
4. 详细描述你的想法：
   - 功能的描述和目的
   - 为什么这个功能很有用
   - 可能的实现方式

### 提交代码

如果你想贡献代码：

#### 1. Fork 项目

点击 GitHub 页面右上角的 "Fork" 按钮。

#### 2. 克隆到本地

```bash
git clone https://github.com/your-username/JMComicReaderProject.git
cd JMComicReaderProject
```

#### 3. 创建分支

```bash
git checkout -b feature/your-feature-name
```

或者修复 Bug：

```bash
git checkout -b fix/your-bug-fix
```

#### 4. 进行修改

- 确保代码符合项目风格
- 添加必要的注释
- 确保没有引入新的 Bug
- 运行测试确保功能正常

#### 5. 提交更改

```bash
git add .
git commit -m "描述你的更改"
```

提交信息格式：
- 功能: `feat: 添加新功能描述`
- 修复: `fix: 修复问题描述`
- 文档: `docs: 更新文档`
- 样式: `style: 代码格式调整`
- 重构: `refactor: 代码重构`
- 测试: `test: 添加测试`
- 杂项: `chore: 其他修改`

#### 6. 推送到 GitHub

```bash
git push origin feature/your-feature-name
```

#### 7. 创建 Pull Request

1. 访问你的 GitHub 仓库页面
2. 点击 "Pull Requests" → "New Pull Request"
3. 选择你的分支
4. 填写 PR 模板：
   - 描述你的更改
   - 关联相关的 Issue
   - 添加截图（如果适用）
5. 提交 PR

## 开发规范

### 代码风格

- Python: 遵循 PEP 8 规范
- JavaScript: 使用 ES6+ 语法
- HTML: 使用语义化标签
- CSS: 使用 BEM 命名规范

### 提交规范

- 使用清晰的提交信息
- 一个提交只做一件事
- 不要提交未完成的代码
- 不要提交敏感信息

### 测试

- 在提交前充分测试你的代码
- 确保不破坏现有功能
- 如有必要，添加相应的测试

## 获取帮助

如果你在贡献过程中遇到问题：

- 查看 [项目文档](README.md)
- 在 Issues 中提问
- 在 PR 中请求代码审查

## 行为准则

- 尊重所有贡献者
- 接受建设性反馈
- 专注于最有益的事情
- 表现出对他人的同理心

## 许可证
